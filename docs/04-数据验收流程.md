# 数据验收流程

**版本：** v1.0  
**编制日期：** 2025.11.18  
**负责人/单位：** AI产业部  
**适用范围：** 日本地区陆上工厂化养殖的南美白对虾数据验收流程与标准

---

## 1. 验收原则

- **客观性**：基于客观标准和规则进行验收，避免主观判断
- **全面性**：覆盖数据质量的所有维度（格式、范围、完整性、一致性）
- **可追溯性**：保留验收记录和审计轨迹，支持问题追溯
- **及时性**：及时完成验收，不延误数据使用

---

## 2. 验收范围

- **新采集数据**：实时采集的传感器数据、图像数据
- **批量导入数据**：历史数据批量导入
- **处理后的数据**：经过ETL/ELT处理的数据
- **关键批次数据**：重要批次的全量数据

## 4. 验收内容

### 4.1 格式校验

- **字段格式**：符合数据库表结构定义
- **数据类型**：符合字段类型要求（INT、DECIMAL、VARCHAR、DATETIME等）
- **编码格式**：UTF-8编码
- **时间格式**：符合DATETIME(3)格式
- **文件格式**：CSV（逗号分隔）、JSON（格式正确）、图像（JPG/PNG/MP4）

**标准**：格式校验通过率 100%

### 4.2 范围校验

- **传感器数据**：数值在合理范围内（参考数据处理标准中的范围定义）
- **图像检测数据**：检测结果在模型训练范围内
- **统计数据**：符合业务逻辑范围

**标准**：范围校验通过率 ≥ 95%

### 4.3 完整性校验

- **必填字段**：所有必填字段必须存在且非空
- **数据完整性**：数据记录完整，无大量缺失
- **元数据完整性**：所有必填元数据字段必须存在且有效
- **关联关系**：外键关联必须存在

**标准**：完整性校验通过率 ≥ 95%

### 4.4 一致性校验

- **单位一致性**：同类数据使用相同标准单位（见数据处理标准）
- **字段一致性**：字段命名和格式统一
- **时间一致性**：时间戳按顺序递增（允许小范围波动），时区转换正确
- **数据一致性**：与历史数据、标准值保持一致

**标准**：一致性校验通过率 100%

### 4.5 处理标准达成检查

- **处理流程**：数据是否经过完整的处理流程（接收→校验→清洗→标准化）
- **处理日志**：处理日志是否完整，处理过程可追溯
- **处理结果**：处理结果是否符合预期

### 4.6 业务逻辑验证

- **业务逻辑符合度**：数据是否符合业务逻辑
- **数据可用性**：数据是否可用于运营分析、模型训练、报表生成

**标准**：业务逻辑符合度 100%

## 5. 验收方式

### 5.1 自动验收

#### 适用场景

- 常规数据验收
- 实时数据验收
- 大批量数据验收

#### 验收流程

```
数据接收 → 基础校验 → 清洗标准化 → 质量规则校验 → 自动验收
```

#### 验收内容

1. **规则引擎校验**
   - 格式校验
   - 范围校验
   - 完整性校验
   - 一致性校验

2. **处理标准达成检查**
   - 处理流程完整性
   - 处理日志完整性
   - 处理结果符合性

3. **元数据完整性检查**
   - 元数据字段齐全性
   - 元数据值有效性
   - 关联关系正确性

#### 验收标准

- **格式校验通过率**：100%
- **范围校验通过率**：≥ 95%
- **完整性校验通过率**：≥ 95%
- **一致性校验通过率**：100%

#### 验收结果

- **通过**：所有校验规则通过，进入人工抽检
- **不通过**：关键校验规则未通过，进入问题处理流程

### 5.2 人工验收

#### 适用场景

- 自动验收不通过的数据
- 关键批次数据
- 异常数据
- 抽检数据（自动验收通过后的抽检）

#### 验收流程

```
自动验收通过 → 人工抽检 → [通过?] → 是：入库/归档 | 否：问题处理 → 重新验收
```

#### 验收内容

1. **数据抽检**
   - 随机抽样检查
   - 基于规则抽样检查
   - 基于风险抽样检查

2. **一致性验证**
   - 与历史数据对比
   - 与标准值对比
   - 与人工标注对比

3. **业务逻辑验证**
   - 数据是否符合业务逻辑
   - 数据是否满足业务需求
   - 数据是否可用于后续分析

#### 验收标准

- **抽检合格率**：≥ 90%
- **一致度**：≥ 85%
- **业务逻辑符合度**：100%

#### 验收结果

- **通过**：抽检合格，数据可用，进入入库/归档
- **有条件通过**：部分问题，但不影响使用，记录问题后通过
- **不通过**：存在严重问题，需要修复后重新验收

## 6. 验收产出

### 6.1 批次验收报告

**报告内容**：
- 基本信息：批次ID、验收时间、验收人员、数据量统计
- 质量指标：完整性、一致性、准确性、及时性、可追溯性指标
- 问题清单：问题类型、数量、严重程度、处理状态
- 处置与结论：问题处置措施、数据修复情况、验收结论、建议事项

**报告格式**：PDF或HTML，中文，保存至少3年

### 6.2 验收台账

**台账内容**：
- 验收记录ID、批次ID、验收时间、验收人员
- 验收方式（自动/人工）、验收结果（通过/有条件通过/不通过）
- 质量指标值、问题数量、处理状态、报告URI

**存储**：数据库表或文档系统，支持按批次、时间、人员查询

## 7. 问题处理

### 7.1 问题分类

- **严重问题**：影响数据可用性（数据大量缺失>20%、关键字段错误、数据范围严重异常），必须修复后才能通过验收
- **一般问题**：影响数据质量但不影响基本使用（少量数据缺失<5%、非关键字段错误、数据范围轻微异常），记录问题，有条件通过
- **轻微问题**：不影响数据使用（格式不规范但不影响解析、元数据缺失但不影响分析），记录问题，建议修复

### 7.2 问题处理流程

```
问题识别 → 问题分类 → 原因分析 → 制定纠正措施 → 执行纠正措施 → 数据修复 → 复验 → [通过?] → 是：问题关闭 | 否：重新分析
```
